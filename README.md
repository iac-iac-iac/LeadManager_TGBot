# Telegram-бот для раздачи холодных лидов с интеграцией в Bitrix24

Система управления холодными лидами для команды менеджеров по продажам (~20 человек) с полной интеграцией в Bitrix24 CRM.

## Статус проекта

**Реализация:** ✅ Завершено

**Версия:** 2.4 (РЕЛИЗ)
**Последнее обновление:** 7 апреля 2026 г.

- ✅ Регистрация менеджеров с подтверждением админа
- ✅ Импорт CSV с защитой от повторов (хеш файла)

## Возможности

### Для менеджеров:
- **Получение лидов**: Выбор сегмента, города, количества (до 200)
- **Статистика**: Детализация по периодам (день/неделя/месяц)
- **Импорт в Bitrix24**: Автоматическое создание лидов с полями
- **Обратная связь**: Отправка тикетов админам (идеи, проблемы, предложения)
- **Мои тикеты**: Просмотр своих обращений к админам

### Для админов:
- **Импорт CSV**: Загрузка файлов с защитой от повторов
- **Проверка дублей**: Верификация через Bitrix24 API
- **Управление сегментами**: Заморозка/разморозка
- **Статистика по менеджерам**: Детальная сводка по каждому
- **Экспорт отчётов**: CSV файлы со статистикой
- **Тикеты менеджеров**: Просмотр и обработка обращений
- **Управление ботом**: Включение/выключение, техобслуживание
- **Рассылка уведомлений**: При запуске/остановке бота
- **Загрузка на Bitrix24 ID**: Импорт лидов на любого пользователя Bitrix24 (без регистрации в боте)
- **Очистка данных**: Удаление старых логов, дублей, импортированных лидов

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка окружения

Скопируйте `.env.example` в `.env` и заполните:

```bash
cp .env.example .env
```

Необходимые параметры:
- `TELEGRAM_BOT_TOKEN` — токен бота от @BotFather
- `BITRIX24_WEBHOOK_URL` — URL входящего вебхука Bitrix24
- `DATABASE_PATH` — путь к SQLite БД (по умолчанию: `./data/leads.db`)
- `UPLOADS_FOLDER` — папка для CSV файлов (по умолчанию: `./uploads`)
- `ADMIN_TELEGRAM_IDS` — Telegram ID админов (через запятую)

### 3. Создание директорий

```bash
mkdir -p data uploads logs
```

### 4. Запуск бота

```bash
# Разработка
python src/bot/main.py

# Production (с логированием)
PYTHONPATH=. python src/bot/main.py
```

### 5. Docker (опционально)

```bash
docker-compose up -d
```

## База данных

### Таблицы

- `leads` — лиды (импортированные из CSV)
- `users` — пользователи (менеджеры и админы)
- `segments` — справочник сегментов (стабильный порядок)
- `segment_locks` — заморозка сегментов/городов
- `logs` — логи событий

### Миграции

Миграции применяются автоматически при запуске:
- v1: Начальная схема (leads, users, logs, segment_locks)
- v2: Дополнительные поля (service_type, stage, phone_source)
- v3: Уникальные индексы
- v4: Таблица segments
- v5: Таблица tickets (обратная связь)
- v6: Таблица bot_status (управление статусом бота)
- v7: Критические индексы производительности (7 новых индексов)

**Применение миграции v7 вручную:**

```bash
# Windows
python apply_migration_v7.py

# Linux/Mac
python src/database/migrations/v7_add_critical_indexes.py
```

## Сценарии использования

### Регистрация менеджера

1. Менеджер нажимает `/start`
2. Вводит Имя и Фамилию
3. Админ получает уведомление
4. Админ подтверждает заявку → вводит Bitrix24 ID
5. Менеджер получает доступ к боту

### Импорт CSV (Админ)

1. Админ загружает CSV файл в папку `uploads/`
2. В боте: "📥 Импорт CSV" → выбрать файл
3. Лиды импортируются в БД (статус: `NEW`)
4. Автоматическая проверка на дубли в SQLite
5. Лиды готовы к выдаче (статус: `UNIQUE`)

### Проверка дублей (Админ)

1. Админ нажимает "🔍 Проверка дублей"
2. Проверяются все лиды со статусом `NEW`
3. Дубли: статус `DUPLICATE`
4. Уникальные: статус `UNIQUE`

### Получение лидов (Менеджер)

1. Менеджер нажимает "📋 Получить лиды"
2. Выбирает сегмент (пагинация 20 на странице)
3. Выбирает город (если есть)
4. Вводит количество (≤200)
5. Показывает доступное количество
6. Подтверждает получение
7. Лиды импортируются в Bitrix24

### Статистика (Менеджер)

1. Менеджер нажимает "📊 Моя статистика"
2. Видит детальную статистику:
   - Сегодня: выдано/импортировано
   - Неделя: выдано/импортировано
   - Месяц: выдано/импортировано
   - Всего импортировано

### Статистика по менеджерам (Админ)

1. Админ нажимает "📊 Статистика по менеджерам"
2. Видит список всех менеджеров
3. Выбирает менеджера
4. Видит детальную сводку:
   - За день/неделю/месяц/полгода/год
   - Всего импортировано
   - ТОП-10 сегментов
5. Может экспортировать в CSV

### Обратная связь (Менеджер)

1. Менеджер нажимает "📮 Обратная связь"
2. Вводит текст сообщения (минимум 10 символов)
3. Подтверждает отправку
4. Получает ID тикета
5. Может просмотреть свои тикеты через "📋 Мои тикеты"

### Обработка тикетов (Админ)

1. Админ нажимает "📮 Тикеты"
2. Видит список тикетов с фильтрацией (Все/Новые/В работе/Закрытые)
3. Выбирает тикет
4. Может ответить менеджеру
5. Может изменить статус (В работу/Закрыть)
6. Менеджер получает уведомление об ответе

### Управление ботом (Админ)

1. Админ нажимает "⚙️ Управление ботом"
2. Видит текущий статус бота
3. Может остановить бота (с указанием причины)
4. Может запустить бота
5. Может включить режим техобслуживания
6. Все пользователи получают уведомления

### Загрузка на Bitrix24 ID (Админ)

1. Админ нажимает "📤 Загрузить на Bitrix24 ID"
2. Вводит Bitrix24 ID пользователя
3. Выбирает сегмент и город
4. Вводит количество лидов (≤200)
5. Подтверждает загрузку
6. Лиды импортируются в Bitrix24 без привязки к менеджеру в боте

**Примечание:** Эта функция позволяет загружать лиды на пользователей, которые не зарегистрированы в боте.

### Уведомления о запуске/остановке

- **При запуске** (`run_bot.bat`): Все активные пользователи получают "✅ Бот запущен!"
- **При остановке** (Ctrl+C): Все активные пользователи получают "⛔ Бот остановлен"
- **При остановленном боте**: Обычные пользователи блокируются с сообщением о техработах

### Управление сегментами (Админ)

1. Админ нажимает "⚙️ Управление сегментами"
2. Выбирает сегмент
3. Замораживает/размораживает
4. Замороженные сегменты не выдаются менеджерам

## Bitrix24 интеграция

### Вебхук

Создайте входящий вебхук в Bitrix24 с правами:
- `crm.lead.*` — работа с лидами
- `crm.duplicate.*` — поиск дублей
- `user.*` — работа с пользователями

### Пользовательские поля

В Bitrix24 создайте поля для лидов:
- `UF_CRM_1745433492760` (Тип услуги) — выпадающий список
- `UF_CRM_1768999878619` (Источник телефона) — строка

### Статусы лидов

| Статус | Описание | Может быть выдан |
|--------|----------|------------------|
| `NEW` | Только что импортирован | ❌ Нет |
| `UNIQUE` | Проверен, дублей не найдено | ✅ Да |
| `DUPLICATE` | Найден дубль в Bitrix24 | ❌ Нет |
| `ASSIGNED` | Выдан менеджеру | ❌ Нет |
| `IMPORTED` | Успешно создан в Bitrix24 | ❌ Нет |
| `ERROR_IMPORT` | Ошибка при импорте | ⚠️ Да |

Логи сохраняются в `logs/bot.log` и выводятся в консоль.

Уровни логирования:
- `INFO` — обычные события
- `WARNING` — предупреждения
- `ERROR` — ошибки
